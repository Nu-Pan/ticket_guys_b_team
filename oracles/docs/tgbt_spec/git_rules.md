# tgbt git 連携ルール

## 概要

- tgbt からの git 操作を tgbt のワークフローの一部として組み込む

## 基本的な考え方

- git 未コミット差分が有る状態で AI にバトンを渡すのは良くない
- AI が変更内容をレビューするための手がかりとして git 履歴を使えるよう、環境を整えたい

## `tgbt` の全体の共通規則

- Codex CLI セッション完了後、未コミット差分は自動的にコミットされる

## plan 起点のサブコマンドたち
    
### `tgbt plan` (新規作成)

- ワークフロー
    1. `tgbt plan` 呼び出し時点で git の未コミット差分は存在するならエラー終了
        - これは、話を簡単にするための保守的な制限
        - 事前にクリーンな状態にするのは人間の責任とする
    2. 新規に作成する plan と対応する git ブランチ（plan ブランチ）を新規に作成・チェックアウトする
    3. 本命処理へ
- 補足
    - plan ブランチの分岐元は default branch の最新コミットとする
    - 実際の plan 作成等の作業はブランチのチェックアウトが済んでから行う

### `tgbt plan` (更新)

- ワークフロー
    1. plan ブランチをチェックアウトしていなければエラー終了
    2. `<repo-root>/oracles` 配下や `<repo-root>/oraclesmemo` 配下などの人間管理責任のファイルに差分が有る場合、それらは tgbt で自動的にコミットする
    3. この時点で git の未コミット差分が残っていればエラー終了
    4. 本命処理へ
- 補足
    - 更新前の plan から plan ブランチ設定を引き継ぐ
        - つまり、新規に作成した plan とその派生 plan はすべて１つの plan ブランチに乗るということ

### `tgbt run`

- ワークフロー
    1. plan ブランチをチェックアウトしていなければエラー終了
    2. `tgbt plan` 呼び出し時点で git の未コミット差分は存在するならエラー終了
    3. 本命処理へ
- 補足
    - この時点で未コミットがある＝やり残しが有るなので `tgbt plan` に差し戻す
    - 反復オーケストレーション中のコードレビュー用のヒントとしてブランチ内のコミット履歴を使えるようにしたい
