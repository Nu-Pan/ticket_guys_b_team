# Knowledge system rules

## 前提

- tgbt では、リポジトリに対する調査を知識システムでラップすることで、トークン消費を抑えることを狙っている
- つまり Codex CLI が知識ステムを利用するように、プロンプトに説明を注入する必要がある
- しかし、知識システム自体の動作にも Codex CLI を使用するため、この知識システム利用指示は固定プロンプトに含めることは出来ない
- そのため、知識システムに関する指示は Knowledge system rules プロンプトブロックとして切り出す

## caller が知識システムではない場合

- リポジトリに対する調査が必要な場合は積極的に知識システムを使うように指示する
- 知識システムの使い方を簡潔に指示する
    - 具体的なコマンド
    - コマンドの出力フォーマット
    - 使い方の具体例

## caller が知識システムである場合

- 知識システムの呼び出しは禁止である事だけを指示する
- その他の情報は一切与えない
