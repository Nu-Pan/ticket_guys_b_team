
# `<tgbt-root>/oracles` の内容を評価するスキル

- 前提として `<tgbt-root>/oracles` は１００％人間の責任でメンテナンスするものであり AI による編集は禁止されている
- しかしながら、人間がメンテナンスするものなので `<tgbt-root>/oracles` に問題（論理的な矛盾など）が存在する可能性は否定出来ない
- これら「`<tgbt-root>/oracles` の問題点」の AI による評価作業はリポジトリローカルなスキルとしてパッケージングされるものとする

# 評価する観点

- 同一ドキュメント内に論理的な矛盾点は有るか？
- 複数の oracles ドキュメント間に論理的な矛盾点は有るか？
- 同じ概念・責務・対象パス・コマンドについて、別概念として誤解されそうな表記ゆれは存在しないか？
- `<tgbt-root>` と `<repo-root>` のような文脈依存のパス表記が、意図と異なる文脈で使われていないか？
- 「AI が編集禁止」「AI が閲覧禁止」「AI が編集可能」など、AI の権限境界に関する記述が他の oracles 記述と衝突していないか？
- スキル間の責務分担が衝突していないか？
- あるスキルの対象外事項が、別のスキルの対象事項として明示されている場合に、その境界が矛盾していないか？
- oracles 本文が参照しているファイル名・ディレクトリ名・スキル名・コマンド名に、明らかな誤記や別 oracles 記述との不整合はないか？
- Markdown の見出し構造・箇条書き・強調表現などが壊れていて、読み手が意味を取り違える状態になっていないか？
- typo、脱字、誤字、明らかな日本語の崩れのような初歩的な問題は存在しないか？

# 評価しない観点

- （`<tgbt-root>/oracles` は不完全であることを前提としているので） `<tgbt-root>/oracles` で言及が漏れている要素の指摘は不要
- `ROUTING.md` の正しさには関知しない（スキル `update-oracles-docs-routing` で面倒を見る）
