

# 概要

- スキル `update-oracles-docs-routing` について述べる
- 本スキルでは `<tgbt-root>/oracles/docs` 配下の `ROUTING.md` の正規化作業を担う

# 参照

- `ROUTING.md` の仕様正本は `oracles/docs/dev_rule/oracles_docs_routing_policy.md` を参照

# 追加プロンプトの対応

- 追加プロンプトで対象ディレクトリの指定があることを前提とする
- 指定がない場合は何もしない

# ワークフロー

1. 機械的なチェック・修正
2. ルーティング本文を AI によってチェック・修正

# 機械的なチェック・修正

- 以下の要件を満たすように、スクリプトによる正規化を行う
    - 指定ディレクトリに `ROUTING.md` が存在している
    - `ROUTING.md` 上の「見出し」と「ファイルシステム上存在するファイル・ディレクトリ」が一致している
    - `ROUTING.md` のフォーマットが仕様どおりになっている

# ルーティング本文を AI によってレビュー・修正

- 指定ディレクトリ直下の `ROUTING.md` が対象
- 「`ROUTING.md` 上のルーティング本文」レビューし、問題があれば修正する
- このチェック・修正作業は AI が行う
- レビュー観点は以下の通り
    - 「ルーティング先の実態」と「`ROUTING.md` 上のルーティング本文」とが整合しているか？
    - ルーティング本文が１～５行程度に収まっているか？
    - ルーティング先の全容を推測可能なルーティング本文になっているか？
    - ルーティング本文はルーティング先の内容を概ねカバーしているか？
    - ルーティング本文が適度に抽象的になっているか？
    - ルーティング本文が「ルーティング先の内容を知らなくても理解できる内容（非自己言及的）」になっているか？
    - typo のような初歩的な問題は存在しないか？
