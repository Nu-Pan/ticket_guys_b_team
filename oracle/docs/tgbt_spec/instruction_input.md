# サブコマンドへの指示文の入力方法

## 方針
- `tgbt plan` などのサブコマンドへ指示文を入力する方法について
- 「標準入力から流し込む」と「エディタが起動して手動入力」の２パターンを用意する

## 標準入力から流し込む
- 引数として末尾に `-` が指定された場合は標準入力から指示文を取り込む
- e.g.
    ```
    cat instruction.md | tgbt plan docs --plan-id <plan id> -
    ```
- こちらはスクリプトなどの機械的な呼び出しに使う想定

## エディタが起動して手動入力

### 概要

- 引数による特別な指示がない場合（デフォルト挙動）はエディタを立ち上げてユーザーに入力を促す
- 環境変数 `$EDITOR` で指定がある場合はそれを使う
- `$EDITOR` が空の場合 code --> vim --> vi の順でエディタ起動を試す

### ワークフロー

- 中継用の「人間指示ファイル」を一時ファイルとして作成
- 人間指示ファイルにテンプレート（後述）を注入
- 人間指示ファイルをエディタで開く
- （人間の操作）エディタで開かれた人間指示ファイルに指示を入力、入力が完了したらエディタをクローズ
- 非コメント部分を人間指示として抽出

## 人間指示ファイルのテンプレート

- 先頭に人間向けの説明をコメントとして注入する
    - 指示文として受け入れ可能なフォーマットの説明
    - 指示文として何を書くべきか？（ベストプラクティス）
- 見出しを注入する
    - tgbt として人間に何を指示してほしいのか＝見出しとみなす
    - 上述のコメントとは別に、見出しで人間の入力を誘導する
    - 見出しは人間によって削除されても良しとする（見出しによる hard gate はしない）

### 人間指示のフォーマット

- 人間指示は markdown 形式で記入する
- コメント
    - `<!-- -->` の形式で記載
    - コメント部分は編集完了後の機械的な後処理で切り落とされる
    - 人間がコメントを記入することは想定していない
- 見出し
    - `#`, `##`, ... の事
    - 使っても良いのは `#` だけで、それ以上のレベル `##`, `###`, ... は使ってはいけない
    - 編集完了後の機械的処理で、自動生成プロンプト全体と整合するように、見出しのレベルの深さは自動調整される
    - 複数のレベルを使うようなしっかりした指示を書きたいのであれば、それは oracle に書くべきという立場を取る
