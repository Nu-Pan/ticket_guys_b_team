
# tgbt における並列・並行処理

- 有る１つのリポジトリの上で複数の tgbt が同時に動くのは禁止
- １つの tgbt 呼び出しの上で複数の Codex CLI 呼び出しが並列に動くのは禁止
- 基本的に並列処理・並行処理は認めず、直列処理とする
- これは、デバッグの簡単さを優先しての割り切りである

# 具体的な原則

- 同じリポジトリ上で既に実行中の tgbt が居る状態で新規に tgbt を実行できてはいけない（排他制御）
- tgbt 内の実装として、AI エージェント呼び出しに並列処理・並行処理を用いてはいけない

# 例外規則

- `tgbt --help` などの、既に実行済み tgbt の挙動に影響を与えない事が明らかな場合のみ、排他制御を行わなくて良い
