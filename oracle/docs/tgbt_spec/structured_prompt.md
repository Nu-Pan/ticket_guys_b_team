# プロンプトの結合原則

## 前提

- AI に実際に渡す指示文は `tgbt` によって動的に構築される
    - e.g. ユーザー入力の指示文＋サブコマンドごとの固定指示文＋Structured Output 用の指示＋…
- この指示文の構築は、実際には指示文断片の結合によって行われる
- AI に渡す指示文は markdown でフォーマットされる

## 結合方法

- 結合対象となる「指示文断片１つ」は「markdown の見出しによるブロック１つ」とする
- 見出しとは `#`, `##`, `###`, ... の事
- この見出しが木構造を形成し、この構造に従って見出しのレベルと結合順序が決定される
- e.g.
    ```json
    {
        "見出し A": "本文 A",
        "見出し B": {
            "見出し B-1": "本文 B-1",
            "見出し B-2": "本文 B-2",
        }
    }
    ```
    これを結合すると、
    ```markdown
    # 見出し A

    本文 A

    # 見出し B

    ## 見出し B-1

    本文 B-1

    ## 見出し B-2

    本文 B-2
    ```
    となる。
