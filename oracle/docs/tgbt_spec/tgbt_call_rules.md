
# tgbt コマンドの複数同時実行についての基本指針

- ある１つのリポジトリ上での tgbt の複数同時実行は、一部例外を除いて原則禁止とする
- この規則により、 tgbt を構成する処理が意味論的には直列であることを保証する
- 実行の直列を保証することで、デバッグの簡単さを優先しての割り切りである

# 基本的な規則

## tgbt の fork-join 的並列処理

- ある１つのリポジトリの上で複数の tgbt が fork-join 的に並列実行するのは禁止とする
- また、１つの tgbt 呼び出しの上で、複数の Codex CLI 呼び出しが　fork-join 的に並列に動くのは禁止とする

## tgbt の stack 的再入処理

- 「tgbt が起動した Codex CLI が tgbt を呼ぶ」ような stack 的な再入処理は原則禁止とする
- ただし、どうしても必要なケースについては、例外的に再入処理を許可する
- e.g. `tgbt knowledge search` サブコマンド
    - Codex CLI が能動的に知識システムにアクセス出来るようにするために、例外的に許可する
    - `tgbt plan` --> Codex CLI --> `tgbt knowledge search` のような再入を想定する

## 副作用を持たないサブコマンド

- `tgbt --help` などの、既に実行中の tgbt の挙動に影響を与えない事が明らかなサブコマンドは、並列実行しても良い

# 実装

## ファイルロックによる排他制御

- `<repo-root>/.tgbt/tgbt.lock` の `flock` により排他制御を行う
- ロックを取得出来なかった場合、ブロックせずに即時に処理を戻す
- このロック取得は tgbt の main 関数の早い段階で行う
- 例外許可ケースに該当する場合はロック獲得を試行しない
- このロック取得規則により `tgbt` の fork-join 実行を防止する

## tgbt 上での Codex CLI の fork-join 並列呼び出し

- python プロセス内同期プリミティブで排他制御を行う
- 並列呼び出しを検知した場合は例外を投げる

## tgbt の stack 再入呼び出しの例外的許可

- 被再入呼び出しではない tgbt 呼び出しのことをルートレベル tgbt と呼ぶことにする
- ルートレベル tgbt は root call id を生成する（uuid）
- ルートレベル tgbt が Codex CLI を起動する際、環境変数 `TGBT_ROOT_CALL_ID` を定義する
- tgbt 起動時に環境変数 `TGBT_ROOT_CALL_ID` が定義されている場合、それを root call id として採用する（生成しない）
- tgbt 起動時に環境変数 `TGBT_ROOT_CALL_ID` が定義されている場合、その tgbt はルートレベルではないと判断する
- 非ルートレベルの場合
    - 呼び出されたのが再入が禁止されているサブコマンドであれば、エラーを出して終了する
    - 呼び出されたのが例外的に再入が許可されているサブコマンドであれば、ロックの取得をせずに処理を続行する
