
# 基本方針

- 実際の運用がデバッグを兼ねることを前提に、テストは最小限に保つ
- テストスイーツとして `pytest` を使う
- テスト以外のチェックとして `pyright` を使う

# `pytest` テスト構成

- `oracle/tests`
    - 人間が100%責任を持つテスト
    - どちらかというと、プログラミング言語で記述する宣託という性格
- `tests/unit`
    - 関数・クラスなどの小さい機能単位のテスト
- `tests/smoke`
    - サブコマンドなど、大きい機能単位のテスト

# smoke テストのアプローチ

- テスト用のトイプロジェクトをサブディレクトリ上に構築し、その上で tgbt を実行する
- この `ticket_guys_b_team` リポジトリ上での `tgbt` の実行（`tgbt` による `tgbt` の自己開発）は禁止

# すべきでないこと

- 単体テスト実行時に live モードを使用してはいけない（テスト実行そのものにトークンを割いてはいけない）
