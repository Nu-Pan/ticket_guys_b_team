# 概念 `oracle`

## 基本的な考え方

- `<repo-root>/oracle` は
    - AI による編集は禁止で、人間が手動で真面目にメンテナンスする
    - 価値判断、非交渉 invariants、外部契約、禁止事項、Don’t Care の境界を書く
    - リポジトリのあらゆる事柄の正本とする
    - コストをかけて、常時クリーンに保つ
- 逆に `<repo-root>/oracle` 以外の箇所に人間は直接手を出さない
- `<repo-root>/docs` は
    - AI によって編集され、人間は直接は触らない
    - 網羅化、具体化、局所ルール展開、実装補助、検索性向上を担う
    - 正本である `<repo-root>/oracle` の派生物
    - ある程度膨らむことは許容する
    - `<repo-root>/oracle` にしか無い情報は許容せず、 AI エージェントは `<repo-root>/docs` だけを見に行けば全ての情報が手に入ることとする

## 狙い

- 人間が手動でメンテナンスしなければならないドキュメント量を小さく保ち、継続メンテナンスが現実的に可能な状態に保つ。
- 人間のフルコミット対象を極小化し、AI によるレバレッジが効くようにする。
- AI がドキュメントをメンテナンスする際、`<repo-root>/oracle` の内容をアンカーとして参照させることで、ドキュメントのドリフティングを抑制する。
- 「人間の意図としてはどうなっても良い（Don't Care）が、実装上は定めたほうが良い仕様」を `<repo-root>/oracle` の外に追いやることで AI の裁量であることを明確化する。

## docs から oracle を参照する方法

- oracle から docs を参照するのは禁止（oracle が正本なので）
- docs 側に orcale の内容の「引用」を書くのは禁止
- docs 側に oracle の見出しへのリンクを書く
- oracle と同じことを docs 内で言いたければ、必ず「参照リンク＋本文コピー」の形式で引用する
- oracle に変更があった場合 docs 側の引用に自動反映される
- 参照リンクが切れている場合、その文章は根本的に書き直し

## tgbt 仕様

oracle にまつわる tgbt の主な仕様は以下の通り

- `tgbt` は、各コマンド経由で人間から与えられた指示を `<repo-root>/oracle` に自動で反映することはしない
- `tgbt eval oracle` サブコマンド
    - `<repo-root>/oracle` に対する評価を行い、その結果を人間にレポートする
    - 主に oracle のみで見た時の「矛盾の有無」「内容の簡略化の余地の有無」「文章構造最適化余地の有無」を評価する
    - ただし oracle の不足点は評価を行わない（oracle の記述は網羅的ではないことを前提とする）
- `tgbt plan` サブコマンド
    - 自然言語指示上で `<repo-root>/oracle` への参照を示すことはあり得るが、それ以上の特別扱いはしない
    - plan の指示と oracle の内容とで矛盾が発生した場合、それは計画書ドラフトに明記される
