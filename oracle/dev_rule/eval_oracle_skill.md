
# oracle の内容を評価するスキル

- 前提として `oracle` は１００％人間の責任でメンテナンスするものであり AI による編集は禁止されている
- しかしながら、人間がメンテナンスするものなので `oracle` に問題（論理的な矛盾など）が存在する可能性は否定出来ない
- これら `oracle` の問題点の AI による評価作業はリポジトリローカルなスキルとしてパッケージングされるものとする

# 評価する観点

- ドキュメント間に論理的な矛盾点は有るか？
- typo のような初歩的な問題は存在しないか？
- 各階層の `ROUTING.md` のファイルリストに過不足はないか？
- そもそも `ROUTING.md` を作り忘れていないか？

# 評価しない観点

- `oracle` で言及が漏れている要素が無いか？
    - `oracle` は不完全であることを前提としているので指摘不要

# ファイルシステム上の実体とルーティング情報の矛盾検出方法

- チェック処理は可能な限りスクリプトで実現する
- e.g. 「`ROUTING.md` 上のファイルリスト」と「実際のファイルシステム上のファイルリスト」はスクリプトで検証する
- 機械的処理を可能とするために `ROUTING.md` は以下のフォーマットに従うものとする
    ```
    # `file_A.md`

    - file_A.md の説明
    - ...

    # `file_B.md`

    - file_B.md の説明
    - ...
    ```
